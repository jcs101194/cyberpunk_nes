PPU Nametable

PPU Nametable - is a 1024 byte area of memory used by the PPU to layout backgrounds. Nametables are

	-32 × 30 tiles
	-Each tile is 8 × 8 pixels
	-So each nametable covers 256 × 240 pixels
	-Hence the diagram below



     (0,0)     (256,0)     (511,0)
       +-----------+-----------+
       |           |           |
       |           |           |
       |   $2000   |   $2400   |
       |           |           |
       |           |           |
(0,240)+-----------+-----------+(511,240)
       |           |           |
       |           |           |
       |   $2800   |   $2C00   |
       |           |           |
       |           |           |
       +-----------+-----------+
     (0,479)   (256,479)   (511,479)


$2000  top-left nametable		# Nametable 0
$2400  top-right nametable		# Nametable 1
$2800  bottom-left nametable	# Nametable 2
$2C00  bottom-right nametable	# Nametable 3

-Despite this, NES usually only has 2 physical nametables, not 4.

Mappers

Mapper - logic inside the cartridge which gives the cpu the needed ROM bank to read

	-The CPU tells the mapper which bank it wants
	-Mapper switches which part of the cartridge is being read
	-The NES itself never knows when this happens


NROM
-No bank switching
-16 KB or 32 KB PRG-ROM
-8 KB CHR-ROM
-Simplest NES cartridge
-Best for learning and very small games

UNROM
-Switchable PRG-ROM banks
-Fixed 8 KB CHR-RAM (graphics loaded at runtime)
-More code space than NROM
-Simple upgrade path from NROM

UNROM-512
-Extended UNROM variant
-Up to 512 KB PRG-ROM
-CHR-RAM only
-Popular for modern NES homebrew projects

CNROM
-Fixed PRG-ROM
-Switchable CHR-ROM banks
-Good for games with many graphics but small code size

MMC1
-Switchable PRG-ROM and CHR-ROM/RAM
-Supports battery-backed saves
-Flexible memory layouts
-Used by early large NES games (Zelda, Metroid)

MMC3
-Advanced PRG-ROM and CHR-ROM bank switching
-Scanline IRQs for mid-frame effects
-Large ROM support
-Used by visually complex NES games (SMB3, Mega Man 3+)

-GTROM
-Modern homebrew mapper
-Very large PRG-ROM support
-Uses CHR-RAM
-Designed for ease of development over historical accuracy

Action53
-Multi-game cartridge mapper
-Used for compilation / multicart projects
-Not typically used for single standalone games


++++++++++++++++++++++++++++
CHR-ROM/RAM

-CHR-ROM - stores graphics on the cartridge and is visible at reset.
-CHR-RAM - starts empty and must be filled by the CPU at runtime, i.e. programatically

-With CHR-ROM, graphics are included in the cartridge and mapped by
hardware to PPU $0000–$1FFF; the CPU never writes tile data at runtime.
